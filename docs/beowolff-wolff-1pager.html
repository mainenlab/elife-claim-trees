Data Layer

Three primary sources are integrated. Artnet's Postgres database provides 20M+ transaction and catalogue records spanning 35 years: sale prices, provenance chains, exhibition histories, and artist biographies. Artsy's behavioral dataset contains 3M user profiles with engagement signals, search sequences, and purchase histories. ArtFacts' knowledge graph encodes gallery representation, institutional affiliations, and critical reception over time. These are resolved into a unified entity graph using four reference ontologies: Iconclass (44K iconographic concepts), Getty AAT (2.5M art and architecture terms), GeoNames for geographic disambiguation, and WordNet for lexical normalization. Entity resolution maps artworks, artists, galleries, collectors, and institutions across all three sources into a single queryable graph containing approximately 41M entities. Every node carries a provenance record indicating which source(s) supplied it and at what confidence.

Feature Extraction

Visual features are extracted by a convolutional neural network architecture calibrated against human preference data from Iigaya et al. (2021, 2023). The model decomposes visual preference into three hierarchical components: low-level statistics — colour, texture, spatial frequency — encoded in layers 1–3; mid-level perceptual structure — composition, form, figure–ground relationships — in layers 4–8; and high-level semantic content — subject matter, iconographic category, stylistic lineage — in layers 9 and above. Each artwork receives a feature vector across all three levels. Semantic and market features — medium, period, provenance chain, price trajectory, institutional history — are extracted from the entity graph. The combined per-work feature space is approximately 2,048 dimensions.

Taste Model

Individual collector taste profiles are fitted as weighted combinations of the three preference components using behavioral data from Artsy. Population-level analysis identifies three stable taste clusters established in Iigaya et al. (2023) that serve as basis vectors; individual profiles are coordinates in this space. Profiles are computed locally — behavioral data is not centralised at any point in the pipeline. Recommendation is nearest-neighbour retrieval in the combined feature–taste space, filtered by availability, price range, and institutional context. Model weights are versioned; each version is logged with a hash of the training corpus at the time of fitting.

Attribution

Each training run logs which data contributed to model performance using a Data Shapley approximation (Beta-Shapley; Kwon & Zou 2022). When a sale occurs following a recommendation, the system traces the recommendation to the model version that generated it, retrieves the Shapley values for that version's training corpus, and distributes a defined fraction of the sale price proportionally across contributing data sources. Distribution is executed via smart contract. Artists and galleries whose data contributed to a recommendation receive automatic, verifiable compensation. The contribution ledger — Filix — records every data deposit, model version, and downstream performance measurement, providing the audit trail that makes Shapley computation tractable and externally verifiable.

Timeline

Week 1: visual preference scores for a sample of 1,000 works using the CNN feature extractor alone — no behavioral data required. Month 6: full three-component feature extraction running on the artnet corpus, taste profiles fitted for Artsy's active collector base, preliminary recommendation outputs available for inspection. Month 12: working recommendation engine operating end-to-end, attribution prototype with documented Shapley values per contributing dataset. Month 18: deployed system with verified attribution payouts on a defined sample of recommendation-driven sales, audit trail complete.
